# Data files for R Journal submission

This folder is split into:

- `raw/`: cleaned, anonymized inputs for AY2420, AY2510, and AY2520.
- `derived/`: tables and plot inputs regenerated from `raw/` by the scripts.

For each semester folder (`raw/ay2420/`, `raw/ay2510/`, `raw/ay2520/`),
`raw/` contains the same five-file schema:

- `students.csv`: anonymized student IDs and workload fields
  (`year`, `past_ta`, `past_gr`).
- `demand.csv`: anonymized course-level demand by role (`TA`, `GR`,
  `E`) with synthetic course codes (`C01`, `C02`, ...).
- `preferences_long.csv`: long-form preference scores by
  `student_id`-`course_code`.
- `manual_totals.csv`: anonymized manual allocation totals by
  student.
- `manual_ta_by_course.csv`: anonymized manual TA units by
  student-course pair.

The raw files intentionally contain no names, emails, intake dates, original
course codes, or workbook metadata.

`derived/` files generated by `scripts/01_build_results.R`:

- `multi_role_dataset_summary.csv`: semester-level dataset sizes and role-demand
  totals.
- `multi_role_objective_comparison.csv`: model and manual objective values under
  the same policy weights.
- `multi_role_objective_terms.csv`: weighted objective-term breakdown for model
  and manual schedules.
- `ay2520_distribution_long.csv`: student-level workload data used for the
  workload-distribution figure.

`ay2520_solver_runtime.csv` is generated separately by
`scripts/03_solver_runtime.R` because it repeats solver runs 30 times for each
open-source solver.

Regenerate derived outputs with:

```r
source("scripts/01_build_results.R")
source("scripts/02_build_plots.R")
source("scripts/03_solver_runtime.R")
```
